# ClawChat 使用指南

本文档面向 OpenClaw 管理员，帮助理解 ClawChat 的定位和使用方式。

---

## ClawChat 是什么

ClawChat 是 OpenClaw 的网页访问界面，帮助管理员将 AI 助手共享给团队成员使用。

团队成员**不需要进入 OpenClaw 界面**，也无需管理员分享 OpenClaw 账号密码。只要通过浏览器访问 ClawChat，就能使用管理员部署的 OpenClaw。

## 基本架构

```
团队成员（浏览器）— 不需要进入 OpenClaw UI
        ↓
ClawChat（网页界面）— 管理用户账号，成员只需使用 ClawChat
        ↓
OpenClaw（AI 核心）— 处理请求、管理权限、建立用户记忆
```

ClawChat 负责：用户账号管理、网页界面、消息转发
OpenClaw 负责：请求处理、权限判定、用户记忆

## 部署后需要做什么

### 1. 创建团队成员账号

管理员在 ClawChat 后台为每个成员创建账号。

### 2. 设置用户角色

| 角色 | 适用人群 | 权限说明 |
|------|---------|---------|
| 普通用户 | 一般成员 | 可发起对话、查询、上传文件，发起写入申请 |
| 部门主管 | 团队负责人 | 普通用户权限 + 审批他人申请 |
| 运维 | 技术负责人 | 主管权限 + 代码修改执行 |

### 3. 决定是否启用用户记忆

OpenClaw 支持为每个用户建立独立记忆空间。**建议启用**，原因如下：

| 好处 | 说明 |
|------|------|
| 个性化回复 | OpenClaw 记住用户的查询偏好、常用术语，回复更精准 |
| 上下文连续 | 用户继续之前的话题，OpenClaw 能直接衔接，不需要重复说明背景 |
| 提升体验 | 越用越懂用户，减少重复解释成本 |

| 注意事项 | 说明 |
|---------|------|
| 数据存储 | 用户历史记录保存在 OpenClaw 部署机器的本地存储中，不上传外部 |
| 隐私告知 | 建议告知团队成员，OpenClaw 会记住他们的使用习惯 |
| 关闭方式 | 如不需要，管理员可在 OpenClaw 端关闭此功能 |

**管理员应根据团队情况自行判断是否启用。**

## 日常工作流

1. **成员登录 ClawChat** — 输入独立账号（不需要进入 OpenClaw）
2. **发起请求** — 对话、查询、上传文件
3. **ClawChat 转发请求** — 附带用户身份标签
4. **OpenClaw 处理** — 根据用户身份和权限返回结果
5. **结果实时推送** — ClawChat 展示给成员

## 权限管理

以下操作需要审批：
- 新增、删除、修改数据记录
- 发送对外邮件
- 执行定时任务变更

普通成员提交申请后，由部门主管在线审批。

## 常见问题

**Q: 如何添加新成员？**
A: 在 ClawChat 后台创建新账号，设置角色即可。

**Q: 如何禁用某个用户的访问权限？**
A: 在 ClawChat 后台删除该账号，或在 OpenClaw 端调整 ACL 配置。

**Q: OpenClaw 如何识别不同用户？**
A: ClawChat 发送消息时会附带用户身份标签，OpenClaw 根据标签识别用户并加载对应记忆。

**Q: 用户上传的文件存在哪里？**
A: 文件存储在 ClawChat 服务端的本地目录，OpenClaw 读取分析后返回结果，不会上传到外部。

**Q: 团队成员需要了解 OpenClaw 吗？**
A: 不需要。成员只需要会使用 ClawChat 的网页界面，所有 OpenClaw 操作对成员透明。
